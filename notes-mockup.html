Collections in Java
Mar 6, 2026 java collections

What are Collections?

A Collection represents a group of objects known as its elements. The Java Collections Framework provides a unified architecture for representing and manipulating collections, enabling them to be manipulated independently of implementation details.

💡 Collections are the backbone of Java data structures. Master these and you'll handle 90% of data manipulation tasks.

Core Interfaces

The framework is built around a set of core interfaces. Each interface defines a different type of collection:

  • List — Ordered collection (sequence). Allows duplicates. e.g., ArrayList, LinkedList
  • Set — Collection that contains no duplicate elements. e.g., HashSet, TreeSet
  • Map — Object that maps keys to values. No duplicate keys. e.g., HashMap, TreeMap
  • Queue — Collection for holding elements prior to processing. e.g., PriorityQueue

See also: [[Functional Interfaces]] for how lambdas work with collections, and [[Stream API Deep Dive]] for processing pipelines.

// Creating and using an ArrayList List<String> fruits = new ArrayList<>(); fruits.add("Apple"); fruits.add("Banana"); fruits.add("Cherry"); // Using Stream API to filter List<String> filtered = fruits.stream() .filter(f -> f.startsWith("B")) .collect(Collectors.toList());

Key Takeaways

  • Understand the Collection hierarchy
  • Know when to use List vs Set vs Map
  • Practice with concurrent collections
  • Learn about Collections.unmodifiableList() patterns
ℹ For thread-safe collections, look into ConcurrentHashMap and CopyOnWriteArrayList from the java.util.concurrent package.
